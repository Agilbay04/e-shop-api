# Store Module Completion Plan

## Current State Analysis

The store module currently has:
- **Model**: `Store` with fields (ID, Name, Description, IsActive, UserID, User relation, Base)
- **Repository**: `StoreRepository` (Create, Update, Delete), `StoreQueryRepository` (FindAll, FindByID, FindByUserID)
- **Service**: `StoreService` with only `CreateStore` method
- **Handler**: `StoreHandler` with only `CreateStore` endpoint
- **Routes**: POST `/api/v1/stores` (protected)

## Required Endpoints

### 1. Get Store Pagination (GET /stores)
- **Access rules**: 
  - Seller users see only their own store
  - Non-seller users (buyer, admin) see all stores
- **Features**: Pagination, search, sorting
- **Response**: Paginated list of stores with user info

### 2. Update Store (PUT /stores/:id)
- **Access**: Only seller who owns the store
- **Fields**: Name, Description (partial update via PUT)
- **Validation**: Cannot update if not owner

### 3. Activate/Deactivate Store (PATCH /stores/:id/status)
- **Access**: Only seller who owns the store
- **Deactivate validation**: Cannot deactivate if store has products in orders with status "pending"
- **Business rule**: Check order_items where store_id = store ID and order status = pending
- **Activate**: Always allowed if user owns store

### 4. Delete Store (PATCH /stores/:id/delete or DELETE /stores/:id)
- **Access**: Only seller who owns the store
- **Delete validation**: Cannot delete if store has products in orders with status "pending" OR "draft"
- **Business rule**: Check order_items where store_id = store ID and order status in (draft, pending)
- **Action**: Soft delete via GORM's Delete (sets DeletedAt)

## Implementation Plan

### Phase 1: DTOs (store_req_dto.go, store_res_dto.go)

**File: `internal/dto/store_req_dto.go`**
- Add `UpdateStoreRequest` struct (Name, Description optional)
- Add `ActivateDeactivateStoreRequest` struct (IsActive bool)

**File: `internal/dto/store_res_dto.go`**
- Add `StoreResponse` struct with full store details including User info
- Add `StorePaginationResponse` (or reuse StoreResponse for items)

### Phase 2: Repository Extensions

**File: `internal/repository/store_query_repository.go`**
- Add `FindAllPagination(req PaginationParam, filter dto.StoreFilter, user CurrentUser) ([]model.Store, int64, error)`
  - Filters: seller sees only own store; non-seller sees all
  - Search by store name or username
  - Sort by name, created_at, is_active
  - Preload User relation

**File: `internal/repository/order_query_repository.go`**
- Add `CountOrderItemsByStoreAndOrderStatus(tx *gorm.DB, storeID string, statuses []string) (int64, error)`
  - Join order_items with orders to count items where order status matches
  - Used for deactivate/delete validation

### Phase 3: Service Implementation

**File: `internal/service/store_service.go`**
- Extend interface with:
  - `GetStoresPagination(req dto.PaginationParam, user dto.CurrentUser) ([]dto.StoreResponse, int64, error)`
  - `UpdateStore(id string, req dto.UpdateStoreRequest, user dto.CurrentUser) (dto.StoreResponse, error)`
  - `ActivateDeactivateStore(id string, req dto.ActivateDeactivateStoreRequest, user dto.CurrentUser) (dto.StoreResponse, error)`
  - `DeleteStore(id string, user dto.CurrentUser) (dto.StoreResponse, error)`

- Implementation details:
  - **GetStoresPagination**: If user is seller, get user's store ID and filter; else return all. Delegate to storeQueryRepo.FindAllPagination.
  - **UpdateStore**: Find store by ID, verify ownership (user.ID == store.UserID), update fields, save via storeRepo.Update, return updated store.
  - **ActivateDeactivateStore**: Verify ownership, if deactivating (IsActive=false) check for pending order items via orderQueryRepo.CountOrderItemsByStoreAndOrderStatus. If count > 0, return error. Else update IsActive.
  - **DeleteStore**: Verify ownership, check for order items with status 'draft' or 'pending' via orderQueryRepo.CountOrderItemsByStoreAndOrderStatus. If count > 0, return error. Else soft delete via storeRepo.Delete (GORM Delete sets DeletedAt).

### Phase 4: Handler Implementation

**File: `internal/handler/store_handler.go`**
- Add `GetStores` method (GET /stores)
  - Bind pagination params from query
  - Call service.GetStoresPagination
  - Use `OkPagination` response
- Add `UpdateStore` method (PUT /stores/:id)
  - Get store ID from params
  - Bind JSON body to UpdateStoreRequest
  - Call service.UpdateStore
  - Use `Ok` response
- Add `ActivateDeactivateStore` method (PATCH /stores/:id/status)
  - Get store ID from params
  - Bind JSON body to ActivateDeactivateStoreRequest
  - Call service.ActivateDeactivateStore
  - Use `Ok` response with status message
- Add `DeleteStore` method (DELETE /stores/:id)
  - Get store ID from params
  - Call service.DeleteStore
  - Use `Ok` response

### Phase 5: Route Registration

**File: `internal/app/router.go`**
- Add protected routes:
  - `GET /stores` -> h.StoreHandler.GetStores
  - `PUT /stores/:id` -> h.StoreHandler.UpdateStore
  - `PATCH /stores/:id/status` -> h.StoreHandler.ActivateDeactivateStore
  - `DELETE /stores/:id` -> h.StoreHandler.DeleteStore

### Validation Rules Summary

- **Ownership**: All operations (except listing) verify store.UserID == currentUser.ID
- **Seller role**: Required for all store operations (enforced in service)
- **Deactivate check**: Count of order_items with pending orders must be 0
- **Delete check**: Count of order_items with draft or pending orders must be 0
